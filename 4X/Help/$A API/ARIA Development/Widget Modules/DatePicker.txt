
ARIA Date Pickers

Expected behaviors: The associated INPUT field should not include readonly, an external triggering element should activate the date picker, the arrow keys should move between calendar cells and the calendar should scroll automatically between months, PageUp/PageDown should switch between months, Alt+PageUp/PageDown should switch between years, Enter should activate the selected date, and pressing Escape or Tab should close the calendar and return focus to the triggering element.

The 4X ARIA DatePicker module automatically configures all required ARIA attributes and focus handling, in strict accordance with the ARIA 1.2 specification.

JavaScript syntax:

$A.setDatePicker({
// Configuration key / value map
});

Programmatic Control

Every Date Picker is registered as an DC object, the ID of which matches the value declared as the "id" property.

var dc = $A('UniqueCalendarId');

// Manually open the calendar.
dc.render();

// Or close it
dc.remove();

// All other 4X API properties and methods can be applied here as well.

Additional Programmatic Control

There are some additional methods which allow altering the Date Picker state after invocation:

* dc.setDayMarked(dc, dateObj, isMarked)

* dc.clearAllMarked(dc)

* dc.setDayDisabled(dc, dateObj, isDisabled)

* dc.setMonthDisabled(dc, dateObj, isDisabled)

* dc.setDayOfWeekDisabled(dc, dateObj, daysOfWeekArray, isDisabled)

* dc.setWeekdaysDisabled(dc, dateObj, isDisabled)

* dc.setWeekendsDisabled(dc, dateObj, isDisabled)

* dc.clearAllDisabled(dc)

* dc.setMonthMessage(dc, dateObj, message)

* dc.clearAllMessage(dc)

* dc.setDate(dc, dateObj)

* dc.presetDate(dc, initialDateObj, minDateObj, maxDateObj)

Straylightping

Straylightping is designed to handle common control types that span multiple pages with similar setup configurations.

The calendars within the Straylight folders are configured using HTML5 "data-" attributes within the markup.

When the Straylight module is loaded, it parses the newly loaded document, recognizes the class "aria-date-picker" on the link or button triggering element, then configures the date picker using these HTML5 attributes.

Available HTML5 attributes for the triggering element:

* data-controls : The id reference for the date picker triggering element to map the corresponding id of the associated input element. The data-controls attribute must be on the link or button triggering element, and its value must match the id value for the associated input element.

Additional HTML5 attributes can be added to enhance functionality by editing the file "/4X/Modules/Straylight".

Required attributes:

* class="aria-date-picker" : The class name that specifies an accessible 4X date picker widget. (Must be on the link or button triggering element.)

Implementation Notes

For more granular access to date string values that exist within each calendar object instance, you can access the internal variables like so:

dc.range.wDays[dc.range.current.wDay].lng = 'Friday'

dc.range[dc.range.current.month].name = 'November'
// or
(dc.range.current.month+1) = numerical month string

dc.range.current.mDay = '30'

dc.range.current.year = '2012'

The returned values will reflect the selected date.
